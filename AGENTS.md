# Agent 行为规范

## 文档同步规则（强制）

**每次修改代码后，必须同步更新项目文档。**

### 触发条件
当发生以下任意一种代码变更时，视为需要更新文档：
- 新增/修改/删除 Vue 组件、页面、路由
- 新增/修改/删除 API 接口或数据模型
- 新增/修改/删除状态管理、工具函数、composables
- 修改构建配置（vite.config.ts、package.json scripts 等）
- 修改 CI/CD 工作流或部署脚本
- 修复 Bug（尤其是根因涉及架构、接口、数据流时）
- 新增/修改单元测试或 E2E 测试

### 必须更新的文档
1. **`docs/智能点菜系统-详细设计说明书.md`**（仓库内受版本控制）
   - 更新「近期变更记录」章节（第 14 节），按日期汇总新增特性、修复、优化
   - 如果修改了架构或目录结构，同步更新「系统架构设计」章节（第 3 节）
   - 如果修改了 API 或数据库，同步更新「API 接口规范」（第 5 节）或「数据库设计」（第 6 节）
   - 如果修改了部署/性能/安全相关代码，同步更新对应章节

2. **`docs/CODE_CHECKLIST.md`**（仓库内受版本控制）
   - 如果是 Bug 修复，按模板追加到「Bug 根因分析库」
   - 如果是代码质量改进，追加到「🟨 建议检查」或「🟥 必须检查」章节
   - 更新「更新记录」表格中的版本和日期

### 执行顺序
代码修改 → 测试通过 → 文档更新 → 部署（如需要）

### 检查方法
在任务完成前，主动问自己：
1. 这次改动是否影响了系统架构、接口、数据模型或部署方式？
2. 如果一个月后团队其他成员看文档，能否从文档中了解到这次变更？
3. CODE_CHECKLIST.md 中是否需要记录新的检查项或 Bug 根因？

如果答案为「是」但文档尚未更新，必须暂停并先完成文档更新。
