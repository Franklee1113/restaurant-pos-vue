## 四、「标记上菜」按钮技术规格

### 4.1 触发条件

仅对 `status === 'cooked'` 的菜品显示按钮。

### 4.2 点击行为

```ts
async function markServed(orderId: string, itemIndex: number) {
  try {
    await OrderAPI.updateOrderItemStatus(orderId, itemIndex, 'served')
    toast.success('已标记上菜')
    await loadData() // 刷新数据
  } catch (err) {
    toast.error('标记失败: ' + (err instanceof Error ? err.message : '未知错误'))
  }
}
```

### 4.3 状态变化

- 点击后 → 菜品 status 变为 `served`
- 页面刷新后 → 该菜品移到列表末尾，淡化显示
- 如果全部菜品 served → Hook 自动推断订单状态为 `dining`
- 如果该桌无其他 cooked 菜品 → 「🔔 待上菜」提示消失

### 4.4 按钮样式

```
[✓ 标记上菜]
```
- 小号按钮（text-xs px-2 py-0.5）
- 绿色边框 + 绿色文字
- 点击后短暂显示「✓ 已上菜」然后消失

## 五、「🔔 有待上菜」筛选逻辑

### 5.1 筛选行为

```ts
const filteredCards = computed(() => {
  let cards = tableCards.value

  // 搜索过滤
  if (searchKeyword.value) {
    cards = cards.filter((c) => c.tableNo.includes(searchKeyword.value))
  }

  // 「有待上菜」筛选
  if (filterCookedOnly.value) {
    cards = cards.filter((c) =>
      (c.order?.items || []).some((item) => item.status === 'cooked')
    )
  }

  return cards
})
```

### 5.2 按钮状态

```
[全部桌台] [🔔 有待上菜]
```

- 「全部桌台」：默认选中，显示所有占用中+待清台桌台
- 「🔔 有待上菜」：选中后只显示包含 cooked 菜品的桌台
- 两个按钮互斥，选中状态有视觉区分（bg-blue-50 text-blue-700）

## 八、数据流（最终版）

```
加载:
  TableStatusAPI.getAllTableStatuses()
    → 筛选 status === 'dining'
    → OrderAPI.getOrders(1, 100, "tableNo IN (...) && status!='settled' && status!='cancelled'")
    → 按 tableNo 分组为 Map<string, Order[]>
    → 构建 TableCard 数组
    → 统计「🟢 N 道待上菜」

实时更新:
  subscribeToOrders('', (record) => {
    // 不过滤 settled/cancelled
    // 前端自行增删改 activeOrders
  })

标记上菜:
  点击 [✓ 标记上菜]
    → OrderAPI.updateOrderItemStatus(orderId, itemIndex, 'served')
    → 后端 Hook 推断订单状态
    → SSE 推送更新
    → 前端自动刷新
```
