## 五、产品代码 Bug 修复记录

测试补测过程中发现并修复了 **2 个产品缺陷**：

### BUG-FIX-001: OrderFormView.vue 折扣比例校验逻辑错位

**缺陷**: `percent` 折扣的范围检查（`> 10` 阻断）被错误地嵌套在 `!parsed.success` 分支内部。当表单其他字段（如 `tableNo`）验证通过时，`percent > 10` 的非法值不会被拦截。

**修复**: 将折扣校验移至 `safeParse` 通过之后独立执行。

**影响文件**: `src/views/OrderFormView.vue`

### BUG-FIX-002: pocketbase.ts HTTP 500 错误消息显示为 `undefined`

**缺陷**: `handleResponse` 在处理 HTTP 500 响应时，如果响应体为 `null` 或无法解析为 JSON，`errorMessage` 会显示为 `HTTP 500: undefined`。

**修复**: 将回退消息改为 `HTTP ${status}${statusText ? ': ' + statusText : ''}`，避免 `undefined` 拼接。

**影响文件**: `src/api/pocketbase.ts`
