## 二、技术风险总结（实施前必须知晓）

### 🔴 已解决的设计层风险

| 风险 | 状态 | 解决方案 |
|---|---|---|
| `pending_clear` 状态不存在 | ✅ 已解决 | 复合判断：`ts.status==='dining' && order.status==='completed'` → 视觉"待清台" |
| 空闲桌台无 `table_status` 记录 | ✅ 已解决 | 以 `settings.tableNumbers` 为全集，`table_status` 为增量 |
| 一桌多单数据覆盖 | ✅ 已解决 | `Map<tableNo, Order[]>` + `currentOrderId` 匹配优先 |

### 🟡 实施层注意事项

| 注意事项 | 说明 |
|---|---|
| **数据一致性** | `table_status` 与 `orders` 可能短暂不一致（如 completed 订单但 ts 仍为 dining），复合判断逻辑已覆盖 |
| **SSE 订阅范围** | 不过滤 settled/cancelled，前端自行移除，避免与 `table_status` 更新竞态 |
| **订单查询优化** | 只查询 `table_status.status==='dining'` 的桌台订单，数据量最小 |
| **空状态处理** | 全店空闲时显示空状态，不展示任何卡片 |

## 三、最终 UI 规格确认

### 3.1 统计栏

```
┌────────────┬────────────┬────────────┬────────────┐
│   🟠 9     │   🟡 3     │   🟢 8     │   ¥3,240  │
│   占用中   │   待清台   │   空闲     │  今日营收  │
└────────────┴────────────┴────────────┴────────────┘
```

- 全部纯展示，无点击交互
- 今日营收 = 所有非 settled/cancelled 订单的 `finalAmount` 累加

### 3.2 筛选栏

```
🔍 搜索桌号...                        [⟳ 刷新]
```

- 仅保留搜索框，按桌号实时过滤
- 无状态筛选（因为只有一种"要看的状态"）

### 3.3 卡片网格

| 设备 | 列数 | 卡片宽度 |
|---|---|---|
| ≥1280px | 3 列 | ~380px |
| ≥1024px | 3 列 | ~310px |
| ≥768px | 2 列 | ~360px |
| <768px | 1 列 | 100% |

### 3.4 单卡片内容规格

```
┌──────────────────────────────────┐
│ 桌号          订单状态标签         │  A2        🔵制作中
│ 订单号                             │  #O202604220012
│ 人数  金额  等待时长               │  👤4人  💰¥268  ⏱️已等12分
│ ───────────────────────────────  │

│   ● 菜名     ×数量  ¥单价         │

│ [查看] [编辑] [打印账单]           │

```

**菜品状态颜色**：
- `pending` / `undefined` → ⚪ 灰色圆点
- `cooking` → 🔵 蓝色圆点 + 脉冲动画
- `cooked` → 🟢 绿色圆点
- `served` → ⚪ 灰色圆点 + `opacity-50` 淡化

**等待时长颜色**：
- `< 10分钟` → 默认灰色
- `≥ 10分钟` → 🟠 橙色
- `≥ 20分钟` → 🔴 红色

## 四、数据流最终确认

```
加载流程:
  TableStatusAPI.getAllTableStatuses()
    ↓
  筛选出 status==='dining' 的桌台
    ↓
  OrderAPI.getOrders(1, 100, "tableNo IN (...) && status!='settled' && status!='cancelled'")
    ↓
  按 tableNo 分组为 Map<tableNo, Order[]>
    ↓
  构建 TableCard 数组

实时更新:
  subscribeToOrders('', (record) => {
    // 不过滤，前端自行判断增删改
  })
```
