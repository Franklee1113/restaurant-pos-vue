## 一、定位调整：从「管理页」到「管理+传菜双角色页」

用户明确：**此页面同时作为传菜员的常用页面**。

这意味着页面需要同时满足两个角色的核心需求：

| 角色 | 核心诉求 | 信息优先级 |
|---|---|---|
| **店长/服务员** | 一扫全店状态，找瓶颈 | 桌台状态 > 等待时长 > 金额 > 菜品 |
| **传菜员** | 快速找到「已做好但未上菜」的菜品 | cooked 菜品 > 桌号 > 人数 > 其他 |

**设计原则**：信息架构以「桌台」为容器，但视觉焦点向 `cooked` 菜品倾斜。

## 三、传菜员视角的业务痛点分析

### 3.1 当前系统的传菜流程缺陷

```
厨房 ──(KDS)──► 厨师点击"已完成" ──► 菜品状态=cooked
                              │
                              ▼  ❌ 没有通知传菜员的渠道
                         传菜员不知道哪桌菜好了
                              │
                              ▼  ❌ 没有"标记已上菜"的UI入口
                         服务员凭记忆/口头通知上菜
```

### 3.2 桌台全景如何弥补

```
厨房 ──(KDS)──► 菜品=cooked ──► 桌台全景实时同步 ──► 传菜员看到 🟢已做好
                                                   │
                                                   ▼  ✅ 视觉高亮 + 桌号清晰
                                              传菜员知道"A3桌铁锅鱼好了"
```

**核心价值**：桌台全景成为连接「厨房出菜」和「传菜上桌」的信息枢纽。

- **占用中 / 待清台 / 空闲**：纯展示
- **🟢 N 道待上菜**：统计全店 `status === 'cooked'` 的菜品总数

  - 这是传菜员的核心入口

### 4.2 新增筛选维度：「有待上菜」

```
[全部桌台] [🔔 有待上菜] [🔍 搜索桌号...]  [⟳ 刷新]
```

- **全部桌台**：显示所有占用中 + 待清台桌台（店长/服务员视角）
- **🔔 有待上菜**：只显示包含 `status === 'cooked'` 菜品的桌台（传菜员视角）

传菜员上班后点击「🔔 有待上菜」，一屏只看出菜任务。

### 4.4 菜品列表排序（传菜员视角）

```ts
const sortedItems = computed(() => {
  const statusPriority = { cooked: 0, cooking: 1, pending: 2, served: 3 }
  return [...order.items].sort((a, b) => {
    return statusPriority[a.status || 'pending'] - statusPriority[b.status || 'pending']
  })
})
```

- 已做好（cooked）排在最前
- 制作中（cooking）其次
- 待制作（pending）再次
- 已上菜（served）最后（淡化）

传菜员一眼看到"该上哪些菜"。

### 4.5 关于「标记已上菜」按钮的关键讨论

**现状**：当前系统没有 UI 入口将 `cooked → served`。

**如果不增加按钮**：
- 传菜员只能"看"，不能"操作"
- 菜品 cooked 后一直显示绿色，无法区分"已上"和"未上"
- 传菜员上菜后没有反馈，页面信息逐渐失真

**如果增加按钮**：
- 传菜员点击「标记上菜」→ 调用 `updateOrderItemStatus(order.id, itemIndex, 'served')`
- 菜品状态变为 served → 淡化显示
- 当全部菜品 served 后，Hook 自动推断订单状态为 `dining`
- **这是现有系统缺失的关键闭环**

**建议**：在 V3 中增加「标记上菜」按钮。

实现方式：
```ts
// 在卡片上为每道 cooked 菜品显示「上菜」按钮
async function markServed(orderId: string, itemIndex: number) {
  await OrderAPI.updateOrderItemStatus(orderId, itemIndex, 'served')
  toast.success('已标记上菜')
}
```

如果用户觉得改动太大，可以分两期：
- **本期**：只显示，不传菜员操作（被动信息展示）
- **下期**：增加「标记上菜」按钮（主动操作闭环）

## 七、数据加载（V3 最终版）

```ts
async function loadData() {
  loading.value = true
  try {
    const tableRes = await TableStatusAPI.getAllTableStatuses()
    tableStatuses.value = tableRes

    const diningTables = tableRes.filter((ts) => ts.status === 'dining')

    if (diningTables.length === 0) {
      activeOrders.value = []
      return
    }

    const tableFilter = diningTables
      .map((t) => `tableNo='${escapePbString(t.tableNo)}'`)
      .join(' || ')
    const filter = `(${tableFilter}) && status!='settled' && status!='cancelled'`
    const orderRes = await OrderAPI.getOrders(1, 100, filter)
    activeOrders.value = orderRes.items
  } catch (err) {
    toast.error('加载失败')
  } finally {
    loading.value = false
  }
}
```
