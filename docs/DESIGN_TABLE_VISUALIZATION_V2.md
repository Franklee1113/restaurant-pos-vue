## 三、统计栏设计

```
┌────────────┬────────────┬────────────┬────────────┐
│   🟠 9     │   🟡 3     │   🟢 8     │   ¥3,240  │
│   占用中   │   待清台   │  空闲(点击)│  今日营收  │
└────────────┴────────────┴────────────┴────────────┘
```

- **占用中**：统计 `displayStatus === 'dining'` 的卡片数
- **待清台**：统计 `displayStatus === 'pending_clear'` 的卡片数
- **空闲（可点击）**：显示空闲桌台数量，点击跳转「新建订单」页面
- **今日营收**：活跃订单 `finalAmount` 累加

空闲数字纯展示，不可点击。

## 四、卡片规格

### 4.1 响应式网格

| 设备 | 列数 | 卡片宽度 | 卡片最小高度 |
|---|---|---|---|
| 桌面端 (≥1280px) | 3 列 | ~380px | 无限制（内容撑开） |
| 笔记本 (≥1024px) | 3 列 | ~310px | 无限制 |
| 平板 (≥768px) | 2 列 | ~360px | 无限制 |
| 手机 (<768px) | 1 列 | 100% | 无限制 |

由于只展示活跃桌台，数量通常 ≤ 15 个，3 列布局完全够用，每个卡片可以做得更大。

### 4.3 菜品行设计

每道菜品显示完整信息：

```
第一行：● 菜名（左对齐）          ×数量  ¥单价（右对齐）
第二行：   状态标签（如 🔵制作中）
第三行：   📝 备注内容（如有）
```

- 菜名左对齐，数量和小计右对齐
- 状态标签在菜名下方缩进对齐
- 备注用灰色小字显示，有备注时才显示
- 已上菜的菜品（`served`）可以淡化显示（opacity-50），或省略不显示

### 4.4 关于已上菜菜品的处理

**已确认：淡化显示**
- `served` 状态的菜品显示为灰色/透明度 50%
- 仍然保留在列表中，服务员可以看到"这道菜已经上了"
- 视觉优先级：待制作 > 制作中 > 已做好 > 已上菜

## 五、新增信息：等待时长

在 V2 中增加「等待时长」显示：

```
⏱️ 已等 12分钟
```

计算方式：
```ts
const waitMinutes = Math.floor(
  (Date.now() - new Date(order.created).getTime()) / 60000
)
```

- 等待超过 **10 分钟** → 文字变橙色
- 等待超过 **20 分钟** → 文字变红色

这是店长/服务员非常关注的指标，直接放在卡片头部。

## 七、数据加载（V2 简化版）

由于只展示活跃桌台，数据加载大幅简化：

```ts
async function loadData() {
  loading.value = true
  try {
    // 1. 加载所有桌台状态
    const tableRes = await TableStatusAPI.getAllTableStatuses()
    tableStatuses.value = tableRes

    // 2. 只保留 dining 状态的桌台
    const diningTables = tableRes.filter((ts) => ts.status === 'dining')

    if (diningTables.length === 0) {
      activeOrders.value = []
      return
    }

    // 3. 只查询这些桌台的订单
    const tableFilter = diningTables
      .map((t) => `tableNo='${escapePbString(t.tableNo)}'`)
      .join(' || ')
    const filter = `(${tableFilter}) && status!='settled' && status!='cancelled'`
    const orderRes = await OrderAPI.getOrders(1, 100, filter)
    activeOrders.value = orderRes.items
  } catch (err) {
    toast.error('加载失败')
  } finally {
    loading.value = false
  }
}
```

空闲桌台不需要查询订单，数据量最小化。

## 八、与 V1 设计的主要差异

| 维度 | V1（全部显示） | V2（仅活跃） |
|---|---|---|
| 展示桌台 | 全部（含空闲） | 仅占用中 + 待清台 |
| 网格列数 | 4~5 列（小卡片） | 3 列（大卡片） |
| 菜品显示 | 最多 4 道，需滚动/折叠 | 完整显示，无需滚动 |
| 单价显示 | 不显示 | 显示每道菜单价 |
| 备注显示 | 不显示 | 显示每道菜备注 |
| 等待时长 | 不显示 | 显示订单等待分钟数 |
| 已上菜菜品 | 全部显示 | 淡化显示（opacity-50） |
| 筛选栏 | 状态筛选 + 搜索 | 仅搜索（因为只有一种"要看的"状态） |
| 空闲桌台操作 | 卡片内「开台」按钮 | 统计栏「空闲」数字可点击跳转新建订单 |
| 空状态 | 空白网格 | 居中提示"所有桌台空闲" |
