## 1. 系统概述

### 1.1 系统定位

智能点菜系统是一套面向餐饮行业的轻量化订单管理解决方案，采用前后端分离架构，支持订单创建、状态流转、菜品管理和数据统计等核心功能。

**2026-04-13 重大更新**：前端已完成从原生 JavaScript SPA 到 **Vue 3 + Vite + Pinia + Tailwind CSS + TypeScript** 的全面重构，并已稳定部署上线。

## 4. 核心业务逻辑

### 4.1 订单状态流转

```
                    ┌─────────────┐
                    │   PENDING   │ ◄── 新建订单默认状态
                    │   (待确认)   │
                    └──────┬──────┘
                           │
           ┌───────────────┼───────────────┐
           │               │               │
           ▼               │               ▼
    ┌─────────────┐        │        ┌─────────────┐
    │   COOKING   │        │        │  CANCELLED  │
    │   (制作中)   │◄───────┘        │   (已取消)   │
    └──────┬──────┘                 └─────────────┘
           │                              ▲
           │                              │
           ▼                              │
    ┌─────────────┐                       │
    │   SERVING   │───────────────────────┤
    │   (待上菜)   │      取消             │
    └──────┬──────┘                       │
           │                              │
           ▼                              │
    ┌─────────────┐                       │
    │  COMPLETED  │───────────────────────┘
    │   (已完成)   │
    └─────────────┘
```

**状态流转规则**（`utils/orderStatus.ts`）

| 当前状态 | 允许流转到 |
|----------|-----------|
| pending (待确认) | cooking, cancelled |
| cooking (制作中) | serving, cancelled |
| serving (待上菜) | completed |
| completed (已完成) | - (终止状态) |
| cancelled (已取消) | - (终止状态) |

### 4.2 订单金额计算（重构后）

```typescript
// utils/security.ts
export class MoneyCalculator {
  static calculateWithDiscount(
    items: CartItem[],
    discountValue: number,
    discountType: 'amount' | 'percent'
  ) {
    const total = items.reduce((sum, item) => {
      return sum + item.price * item.quantity
    }, 0)

    let discount = 0
    if (discountType === 'percent') {
      discount = Math.round(total * (discountValue / 100) * 100) / 100
    } else {
      discount = discountValue
    }

    const final = Math.max(0, Math.round((total - discount) * 100) / 100)
    return { total, discount, final }
  }
}
```

```nginx
server {

    server_name _;

    # 前端静态文件 (Vue 3 SPA)

            add_header 'Access-Control-Allow-Headers' 'DNT,User-Agent,X-Requested-Since,Cache-Control,Content-Type,Range,Authorization';
```

## 8. 安全设计

### 8.1 认证机制

1. **JWT Token 认证**
   - 登录成功后获取 Token
   - Token 存储于 `localStorage`（由 `auth.store.ts` 统一管理）
   - 每次 API 请求通过 PocketBase SDK 自动携带 Bearer Token
   - 路由守卫在页面切换时自动校验登录态

2. **Token 刷新**
   - PocketBase 默认 Token 有效期管理
   - 401 响应触发自动登出并跳转登录页

### 8.2 数据安全

1. **XSS 防护** (`utils/security.ts`)
   - 所有用户输入和动态内容通过 `escapeHtml()` 转义
   - 避免将不可信数据直接注入 DOM

2. **金额安全** (`MoneyCalculator`)
   - 使用整数/定点数思维处理金额计算
   - 避免 JavaScript 浮点数精度问题导致的价格偏差

3. **PocketBase 权限规则**
   - 当前配置：`listRule: null` (无限制)
   - 生产环境建议配置详细的 `listRule` / `updateRule`

### 8.3 Nginx 安全配置

- `X-Frame-Options: SAMEORIGIN` 防止点击劫持
- `X-Content-Type-Options: nosniff` 防止 MIME 嗅探
- CORS 跨域配置限制在 `/api/` 路径
